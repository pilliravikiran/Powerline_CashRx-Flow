Opt-out clarification: Pharmacies that opt out of RxAdvantage do so via RedSail RASE enrollment — the opt-out is configured on the RedSail/PowerLine side, not in PrimeRx. PrimeRx PMS sends the same standard B1 claim it would send for any Cash Rx (BIN 028249, Submission Type 0, PCN, Person Code 001, Relationship 01). PrimeRx does not add an EW/EX Override for opted-out pharmacies — Override '01' is reserved for the per-Rx discount-removal flow. RedSail recognises the pharmacy's opt-out enrollment and simply does not return any Auto Discount (J5 = 0, no RXADV), so the CASH Transmission Result popup never fires, but Pay Records / TR-Log / TR-String are still captured for audit and data aggregation.

How to read the master diagram:
  1. Entry — top of chart: Save action, Defer-before-Save, and the eligibility / opt-out branches (note the distinct B1 variants: standard, opt-out, or aggregation-only).
  2. Initial B1 transmission — all three variants converge into one PowerLine call, which either succeeds or goes into the Switch-Down path.
  3. Response handling — F4/FQ/J5 detection determines popup vs no-popup path.
  4. Post-Billed user actions — every user action in Edit Rx funnels into the B2 reversal pipeline (with per-trigger prompt / silent behaviour).
  5. Reversal — central B2 send, with switch-down BLOCK handling.
  6. Post-reversal branching — Unbilled / Returned / Deleted / Deferred terminal states, plus the auto-rebill loop for toggle flows.
  7. Rebill — persisted Apply Auto Discount intent decides standard B1 vs B1+Override '01'; each has its own switch-down behaviour (allow vs block).
